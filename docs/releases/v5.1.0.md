# Kap v5.1.0

Tag: `v5.1.0`

Kap v5.1.0 is a community-maintained release of the Kap macOS screen recorder focused on the editor, screenshot workflow, packaging safety, updater compatibility, and Kap branding in packaged builds.

## Downloads

- Apple Silicon - M1, M2, M3, M4, and newer: download `Kap-5.1.0-mac-arm64.dmg`
- Intel Mac: download `Kap-5.1.0-mac-x64.dmg`

Open **Apple menu -> About This Mac** if you are unsure which architecture your Mac uses. Macs that show **Chip** are Apple Silicon. Macs that show an Intel **Processor** use the x64 build.

## Highlights

- Timeline editor for splitting, trimming, deleting, copying, pasting, cutting, duplicating, seeking, playback, and five-second skip controls.
- Freeze-frame clips inserted at the selected frame with adjustable still duration.
- Clip adjustments for speed, brightness, contrast, and saturation with timeline-aware export rendering.
- Screenshot mode from the capture overlay, including direct Command+C clipboard capture.
- Persistent cropper bounds and safer cropper interaction reset across open, close, Escape, blur, and display-change paths.
- Repaired updater checks against the latest GitHub release and architecture-specific DMG assets.
- Refined Kap branding for packaged application metadata, helper apps, Dock behavior, notifications, and development launch metadata.
- README updated for Kap v5.1 users and contributors.

## Updating

Kap v5.0 users can use **Check for Updates...** from Kap after the `v5.1.0` GitHub Release is published. The updater checks the latest public GitHub release and opens the correct `mac-x64` or `mac-arm64` DMG asset when available.

This updater does not silently install updates. Users still download the DMG, open it, and drag `Kap.app` into Applications.

## Known Issues

- Signing and notarization depend on Apple Developer credentials in the release environment.
- macOS may require manual approval in **System Settings -> Privacy & Security** for unsigned or unnotarized builds.
- Runtime verification on Apple Silicon requires Apple Silicon hardware or a matching CI runner.
- System audio capture requires a user-installed loopback audio device because macOS does not expose output audio as a normal microphone.

## Verification Checklist

Record final checksums, artifact sizes, code signing status, notarization status, and manual runtime verification results in the GitHub Release notes after packaging the final DMGs.
